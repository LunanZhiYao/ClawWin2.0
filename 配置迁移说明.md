# 配置迁移功能说明

## 功能概述

从 v3.2.7 开始，ClawWin 内置了智能配置迁移功能，可以自动处理版本升级时的配置不兼容问题。

## 三层防御机制

### 第 1 层：启动前校验（90% 覆盖率）

**触发时机**：每次启动 Gateway 之前

**工作流程**：
1. 读取 `~/.openclaw/openclaw.json` 配置文件
2. 检查 JSON 格式是否正确
3. 检查配置版本是否匹配
4. 自动迁移配置，保留以下用户数据：
   - Gateway Token（认证令牌）
   - Models（所有模型配置，包括 API Key）
   - Agents（所有 Agent 配置）
   - Channels（Telegram/Feishu 等聊天工具配置）
   - Pairings（设备配对信息）
   - Settings（其他设置）

**用户体验**：
- ✅ 90% 情况：自动迁移成功，用户无感知
- ⚠️ 9% 情况：弹窗提示"配置不兼容，是否重置？"
- ❌ 1% 情况：提供"查看日志"选项供技术用户排查

### 第 2 层：运行时监控（9% 覆盖率）

**触发时机**：Gateway 进程异常退出时

**工作流程**：
1. 收集 Gateway 运行日志（最近 100 条）
2. 检测日志中是否包含配置相关错误关键词：
   - `validation error`
   - `invalid config`
   - `config parse error`
   - `JSON parse error`
   - `schema validation failed`
   - `configuration error`
   - `config file corrupted`
3. 如果检测到配置错误，自动弹窗提示用户重置配置
4. 重置前自动备份旧配置

**用户体验**：
- 弹窗提示："检测到配置相关错误，尝试自动修复..."
- 提供三个选项：重置配置 / 查看日志 / 退出应用

### 第 3 层：重启失败提示（1% 覆盖率）

**触发时机**：Gateway 重启失败时

**工作流程**：
1. 捕获重启失败异常
2. 弹窗提示用户可能的原因和解决方案

**用户体验**：
- 弹窗标题："网关重启失败"
- 建议操作：
  1. 完全退出应用（右键托盘 → 退出）
  2. 等待 5 秒
  3. 重新启动应用
- 提供三个选项：退出应用 / 重置配置 / 取消

## 配置备份机制

每次重置配置前，系统会自动备份旧配置：

**备份位置**：`~/.openclaw/openclaw.json.backup-时间戳`

**备份文件名示例**：
- `openclaw.json.backup-2026-03-09T15-30-45`

**恢复方法**：
1. 打开 `C:\Users\你的用户名\.openclaw\` 目录
2. 找到最新的备份文件
3. 删除当前的 `openclaw.json`
4. 将备份文件重命名为 `openclaw.json`
5. 重启 ClawWin

## 迁移保留的数据

### ✅ 会保留的数据

1. **Gateway Token**：认证令牌，无需重新登录
2. **Models**：所有模型配置
   - provider（提供商）
   - modelName（模型名称）
   - apiKey（API 密钥）⭐ 重要
   - baseURL（API 地址）
   - maxTokens（最大 Token 数）
   - temperature（温度参数）
3. **Agents**：所有 Agent 配置
   - id、name（标识和名称）
   - model（使用的模型）
   - systemPrompt（系统提示词）
   - tools（可用工具列表）
4. **Channels**：聊天工具配置
   - Telegram Bot Token ⭐ 重要
   - Feishu App ID/Secret ⭐ 重要
5. **Pairings**：设备配对信息
6. **Settings**：其他用户设置

### ❌ 会丢失的数据（仅在重置时）

- 聊天历史记录（存储在其他位置，不受影响）
- 临时会话状态

## 技术实现

### ConfigMigrator 类

**位置**：`electron/config-migrator.ts`

**核心方法**：
- `validateAndMigrate()`：主入口，校验并迁移配置
- `migrateConfig()`：执行迁移，保留用户数据
- `sanitizeModel()`：清理模型配置，只保留兼容字段
- `sanitizeAgent()`：清理 Agent 配置
- `backupConfig()`：备份配置文件
- `promptUserToReset()`：提示用户重置配置
- `detectConfigError()`：检测日志中的配置错误

### GatewayManager 集成

**位置**：`electron/gateway-manager.ts`

**集成点**：
1. `doStart()`：启动前调用 `validateAndMigrate()`
2. `process.on('exit')`：进程退出时调用 `handleGatewayCrash()`
3. `restart()`：重启失败时调用 `showRestartFailureDialog()`

## 用户常见问题

### Q1: 升级后提示"配置不兼容"怎么办？

**A**: 点击"重置配置"按钮，系统会自动备份旧配置并重置。重置后需要重新配置模型和 API Key。

### Q2: 重置配置会丢失什么？

**A**: 会丢失所有模型配置（API Key）、Agent 配置、聊天工具配置。但旧配置会自动备份，可以手动恢复。

### Q3: 如何恢复备份的配置？

**A**:
1. 打开 `C:\Users\你的用户名\.openclaw\` 目录
2. 找到 `openclaw.json.backup-时间戳` 文件
3. 删除当前的 `openclaw.json`
4. 将备份文件重命名为 `openclaw.json`
5. 重启 ClawWin

### Q4: 为什么会出现配置不兼容？

**A**: 主要原因：
- 从旧版本升级后，配置文件结构发生变化
- Gateway 使用严格的 JSON Schema 校验
- 字段名称、类型或结构不匹配

### Q5: 如何避免配置不兼容？

**A**:
- 升级前完全退出旧版本（右键托盘 → 退出）
- 等待 5 秒后再安装新版本
- 系统会自动迁移配置，大部分情况无需手动处理

## 开发者说明

### 添加新的配置字段

如果需要在配置中添加新字段，请在 `ConfigMigrator.migrateConfig()` 中添加相应的迁移逻辑：

```typescript
// 示例：添加新的 features 字段
if (oldConfig.features) {
  newConfig.features = oldConfig.features
}
```

### 修改配置结构

如果需要修改配置结构（如重命名字段），请在 `migrateConfig()` 中添加转换逻辑：

```typescript
// 示例：将 oldField 重命名为 newField
if (oldConfig.oldField) {
  newConfig.newField = oldConfig.oldField
}
```

### 测试迁移逻辑

1. 准备一个旧版本的 `openclaw.json` 文件
2. 启动新版本 ClawWin
3. 检查日志中的迁移信息
4. 验证配置是否正确迁移
